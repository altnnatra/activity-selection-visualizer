📊 CLO-1 · Complexity Analysis — Why O(n log n)? ▼
Best Case
O(n log n)
Worst Case
O(n log n)
Average
O(n log n)
Space
O(n)

The algorithm has two phases. Sorting by finish time costs O(n log n) — this dominates. The Greedy loop is just O(n): one pass through the sorted list. Together: O(n log n) + O(n) = O(n log n).

Why is sorting O(n log n)? Any comparison-based sort needs at least log₂(n!) ≈ n·log(n) comparisons to distinguish all n! orderings. Merge Sort achieves this exactly.

Max n = 100
▬ O(n log n) — Activity Selection ▬ O(n²) — Brute Force ▬ O(n) — Greedy loop only
🔀 CLO-2 · Role of Sorting — Why Finish Time, Not Start Time? ▼

Sorting by finish time is the critical first step. Without it, the greedy algorithm can pick a long activity early and block all shorter ones.

⚠️ Counterexample — Sort by start time instead:

Activities: [1,10], [2,3], [4,5], [6,7]
• Sort by start → greedily pick [1,10] → blocks everything → 1 selected ✗
• Sort by finish → pick [2,3] → [4,5] → [6,7] → 3 selected ✓
🧠 CLO-3 · Greedy Paradigm — Why Is It Provably Optimal? ▼
🎯 Greedy Choice Property:
Always selecting the activity with the earliest finish time is locally optimal AND leads to a globally optimal solution. No backtracking needed — every greedy choice is safe.
🔄 Exchange Argument (Proof Sketch):
Suppose optimal solution OPT doesn't include greedy's first choice a₁ (earliest finish). OPT must start with some other activity aₓ. Since f(a₁) ≤ f(aₓ), swap aₓ → a₁. Everything else in OPT still fits because a₁ ends no later. So greedy is never worse.
🧩 Optimal Substructure:
After selecting a₁, the remaining problem is identical: find max compatible activities starting at or after f(a₁). Each greedy step reduces the problem optimally.
🔢 Greedy vs Brute Force:

Brute Force: check all 2ⁿ subsets → O(2ⁿ)
Greedy: sort + one pass → O(n log n)

n=20: Brute = 1,048,576 subsets. Greedy ≈ 86 operations.
n=50: Brute = 1,125,899,906,842,624. Greedy ≈ 282 operations.
📖 Overview & Fully Worked Example ▲

The Activity Selection Problem: given n activities on a shared resource (e.g. a lecture hall), each with start time sᵢ and finish time fᵢ, find the maximum number of mutually compatible (non-overlapping) activities. Two activities i,j are compatible if fᵢ ≤ sⱼ or fⱼ ≤ sᵢ.

Fully Worked Example — 6 activities:
Activity123456
Start s1305812
Finish f45671116
  1. Sort by finish time: [1,4], [3,5], [0,6], [5,7], [8,11], [12,16] ← already sorted
  2. Select activity 1 [s=1, f=4]. A = {1}, j = 1
  3. i=2: s₂=3 ≥ f₁=4? NO → REJECT activity 2
  4. i=3: s₃=0 ≥ f₁=4? NO → REJECT activity 3
  5. i=4: s₄=5 ≥ f₁=4? YES → SELECT activity 4. A = {1,4}, j = 4
  6. i=5: s₅=8 ≥ f₄=7? YES → SELECT activity 5. A = {1,4,5}, j = 5
  7. i=6: s₆=12 ≥ f₅=11? YES → SELECT activity 6. A = {1,4,5,6}

✓ Result: 4 activities {1,4,5,6} — the maximum possible.
